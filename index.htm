On HTML's peculiar syntax

Who came up with HTML's peculiar syntax? It just goes over my head how it might possible be considered convenient to type every keyword twice. I know, compared with something like C it is easier to see where every scope ends, but still, it seems like a massive waste of keystrokes to me, especially compared to languages like Markdown or org-mode.

The plain text look

I like the look of plain text. It simply looks "clean". There are some things to consider when writing in plain text, though:
  • Setting margins may require manual work
  • Bullet lists definitely require manual work
  • Adding empasis is harder, by lack of such conveniences as bold or italic text
That's why I have decided to merely give this HTML document a plain text look, instead of using actual plain text. For this, all I had to do was setting a monospace font, and using margins and the "max-width" property in CSS to mimic the relatively short lines commonly used in plain text documents.
